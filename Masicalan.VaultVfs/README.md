# Masicalan Vault Virtual File System
Mashiro Tamane

### イントロダクション
**Masicalan Vault Virtual File System** (Masicalan Vault VFS; MVVFS)は，
Masicalan のスクリプトファイル(.masis)を格納するための暗号化仮想ファイルシステムです．
Masicalan は .NET アプリケーションのマクロ記述言語となることを目的としていますが，
マクロ機能最大の懸念点の一つは，不正なマクロスクリプトを挿入されることによるセキュリティリスクです．<br>
そこで，Masicalan Vault VFS は，マクロスクリプトファイルを暗号化された仮想ファイルシステムに格納することによって，
ユーザーがテキストエディタでスクリプトを書き換えたり，
外部のプログラムが安易にスクリプトを書き換え，挿入することを防ぐことを目的として開発されています．

### 仕様
Masicalan Vault VFS は，ZIPファイルを Windows の DPAPI を使って暗号化して，
擬似的な暗号化ファイルシステムを実現しています．
ZIPファイルのエントリをそのままディレクトリ構造として利用しています．
Masicalan Vault VFS 内には```scripts/```というルートディレクトリがあり，
ファイルシステムはこのルートディレクトリに対する相対パス，
あるいは```scripts/sample.masis```，```scripts/subdir/sample.masis```のような形による絶対パスでファイル・ディレクトリ構造を表現します．<br>
暗号化には Windows の DPAPI を利用しています．
Masicalan Vault VFS は F# + .NET で実装されており，
```System.Security.Cryptography```名前空間を利用しています．
```DataProtectionScope```は```CurrentUser```で暗号化してます．
すなわち，ログインしているユーザー固有の情報に基づいて暗号化が施されます．<br>
Masicalan Vault VFS では，各スクリプトファイルに権限属性を付与することができます．
この権限属性とはファイルの編集及び実行に関する属性であり，以下の3種類を設定可能です．
- ReadOnly (読み取り専用)
- Editable (編集可能)
- Executable (実行可能)
<br>
VFS では，スクリプトファイルの追加・編集時にハッシュ値を計算して記録しており，
ファイルから計算したハッシュ値と記録されているハッシュ値が異なる場合，読み取りと編集を拒否する(例外スロー)ようになっています．

### 開発環境
- Microsoft Windows Insider Preview Exprimental
- Visual Studio 2026
- .NET SDK 10.0

### 注意点
Masicalan Vault VFS は外部のプログラムが安易にスクリプトを書き換えたり不正に挿入することを防止することを目的としています．
しかし，暗号化には Windows DPAPI を利用しており，
エントロピーの管理方法によっては同様に DPAPI を使って外部プログラムが VFS を編集することが可能である場合があります．
**あくまで簡易的なファイルI/Oによるスクリプトファイルのインジェクション攻撃を最低限防ぐことを意図したものであり，
あらゆる脅威に対する保護を目的としたものではありません**．<br>
VFS はユーザー情報に基づいて暗号化が施されます．
したがって，**ユーザーを作り直した場合や，別のユーザーでログインして VFS を開こうとした場合，失敗します**．
また，**何らかの理由でユーザー情報が破損した場合やドメインアカウントの管理者によるパスワード強制リセットの一部の場合では
VFS の復号が不可能になる可能性もあります**．
Masicalan Vault VFS を利用するアプリケーション開発者は，スクリプトファイルのバックアップ機能あるいはエクスポート機能などを適宜実装する必要があります．
